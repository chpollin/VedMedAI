# CLAUDE - Arbeitsregeln

## PFLICHTEN

### Dokumentation pflegen
Alle Sessions in WORKING.journal.md kompakt und präzise dokumentieren
Alle Erkenntnisse in docs/knowledge/ ablegen
Bei jeder Session WORKING.journal.md aktualisieren

### Knowledge Base
Ordner docs/knowledge/ kontinuierlich erweitern
Neue Dateien nur bei relevantem Wissensgewinn erstellen
Bestehende Dateien aktualisieren statt neue anzulegen

### Arbeitsjournal
WORKING.journal.md bei jeder Session aktualisieren
Format: Datum, Aufgabe, Ergebnis, Erkenntnisse
Kompakt halten, keine Wiederholungen

## VERBOTE

### Formatierung
Niemals Emojis verwenden
Niemals bold verwenden (kein doppeltes Sternchen)
Nur einfache Markdown-Formatierung

### Code
Keine Emojis in Kommentaren
Keine dekorativen Elemente
Nur sachliche technische Dokumentation

### Kommunikation
Sachlich bleiben
Keine emotionalen Formulierungen
Keine übertriebene Höflichkeit

## STIL

### Text
Kurz und präzise
Technisch korrekt
Ohne Füllwörter

### Listen
Stichpunkte statt Fließtext
Klare Hierarchien
Keine Verschachtelungen über 3 Ebenen

### Überschriften
Nur # Raute für Hierarchie
Keine Unterstreichungen
Keine Nummerierungen in Überschriften

## ARBEITSWEISE

### Sessions
Vor jeder Aufgabe WORKING.journal.md lesen
Erkenntnisse aus vorherigen Sessions nutzen
Dopplungen vermeiden

### Code
Python-Scripts in pre-processing/
Analysen dokumentieren
Ergebnisse in docs/knowledge/

### Daten
Rohdaten in data/ unverändert lassen
Verarbeitete Daten dokumentieren
Strukturen in DATA.md pflegen

## STRUKTUR

### Ordner
docs/knowledge/ - Wissensdokumentation
pre-processing/ - Datenaufbereitung
data/ - Rohdaten (read-only)

### Dateien
CLAUDE.md - Diese Datei, Arbeitsregeln
WORKING.journal.md - Arbeitsjournal aller Sessions
DATA.md - Datendokumentation
README.md - Projektübersicht

### Benennung
Kleinbuchstaben bevorzugen
Bindestriche statt Unterstriche
Sprechende Namen

## QUALITÄT

### Präzision
Exakte Definitionen
Messbare Werte
Nachvollziehbare Quellen

### Konsistenz
Einheitliche Terminologie
Gleiche Strukturen
Wiedererkennbare Muster

### Vollständigkeit
Alle relevanten Informationen
Keine Lücken
Verweise auf Zusammenhänge

## UPDATES

### Bei Code-Änderungen
Pre-processing README aktualisieren
Neue Scripts dokumentieren
Abhängigkeiten pflegen

### Bei Daten-Änderungen
DATA.md sofort aktualisieren
Strukturänderungen dokumentieren
Neue Kategorien erfassen

### Bei Sessions
WORKING.journal.md Eintrag anlegen
Wichtige Erkenntnisse in knowledge/
Code in pre-processing/
