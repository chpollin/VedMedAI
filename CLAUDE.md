# CLAUDE - Arbeitsregeln

## VERBOTE
- Keine Emojis (Text, Code, Kommentare)
- Kein Bold (doppelte Sternchen)
- Keine emotionalen Formulierungen
- Keine dekorativen Elemente

## STIL
- Kurz, präzise, technisch korrekt
- Stichpunkte statt Fließtext
- Klare Hierarchien (max 3 Ebenen)
- Sprechende Namen (kleinbuchstaben, bindestriche)

## DOKUMENTATION
- Jede Session in PROJECT-JOURNAL.md (Format: Datum, Aufgabe, Ergebnis)
- Erkenntnisse in docs/knowledge/
- Bestehende Dateien aktualisieren statt neue anlegen
- Exakte Definitionen, messbare Werte, nachvollziehbare Quellen

## STRUKTUR
- docs/knowledge/ - Wissensdokumentation
- pre-processing/ - Datenaufbereitung (Python-Scripts)
- data/ - Rohdaten (read-only)

Dateien:
- CLAUDE.md (Root) - Arbeitsregeln
- PROJECT-JOURNAL.md (knowledge) - Session-Protokoll
- DATA.md (knowledge) - Datendokumentation

## UPDATES
- Code: pre-processing/README.md aktualisieren
- Daten: DATA.md sofort aktualisieren
- Sessions: PROJECT-JOURNAL.md vor jeder Aufgabe lesen
